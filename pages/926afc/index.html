# 1. 需求分析

# 1.1 需求规格说明书

包含但不限于以下内容：

  1. 范围
  2. 引用文件
  3. 需求：需求概述、需求规格（软件总体功能/对象结构、软件子系统功能/对象结构）、功能需求、接口需求、数据需求、保密需求、硬件网络环境需求
  4. 合格性规定：演示、测试、分析等形式
  5. 尚未解决的问题：如需要，可说明软件需求中尚未解决的遗留问题
  6. 注解

# 1.2 需求确认

针对需求规格说明书，提交带有用户签字的确认文件。

# 2. 设计开发

# 2.1 设计方案

包含但不限于以下内容：

  1. 信息资源规划
  2. 应用系统设计
  3. 网络系统设计
  4. 数据处理和存储系统设计
  5. 安全系统设计
  6. 其他系统设计（终端、备份、运维）

# 2.2 概要设计说明书

可与详细设计整合在一个标题下，包含但不限于以下内容：

  • 1 引言
    • 1.1 编写目的
    • 1.2 背景
  • 2 总体设计
    • 2.1 运行环境

    • 2.5尚未解决的问题
  • 3 接口设计
    • 3.1用户接口
    • 3.2外部接口
    • 3.3内部接口
  • 4 运行设计
    • 4.1运行模块组合
    • 4.2运行控制
    • 4.3运行时间
  • 5 系统数据结构设计
    • 5.1逻辑结构设计要点
    • 5.2物理结构设计要点
    • 5.3数据结构与程序的关系
  • 6 系统出错处理设计
    • 6.1出错信息
    • 6.2补救措施
    6.3系统维护设计
